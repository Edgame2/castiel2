## [Unreleased]

### Changed
- **README:** Prerequisites and Dependencies now describe API base URL via `NEXT_PUBLIC_API_BASE_URL` instead of hardcoded port.

### Added

- **W11 – Super Admin: Tenant ML Config and Sales Methodology (Plan W11, Feedbacks and Recommendations):** Admin dashboard links for "Tenant ML Config" and "Sales Methodology". `/admin/tenant-ml-config`: GET/PUT `/api/v1/tenant-ml-config` via gateway (risk-analytics); form for risk tolerance (overall, auto-escalation threshold), decision preferences (autoMarkHot, autoCreateTasks, requireApprovalForActions), model preferences (preferIndustryModels, abTestingEnabled, minConfidenceThreshold). `/admin/sales-methodology`: GET/PUT `/api/v1/sales-methodology` via gateway; form for methodology type (MEDDIC, MEDDPICC, etc.) and JSON editors for stages, requiredFields, risks. Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: Risk Catalog tenant catalog view:** Admin dashboard link "Risk Catalog". `/admin/risk-catalog`: GET `/api/v1/risk-catalog/tenant-catalog` via gateway (risk-catalog); optional industry and stage query filters; displays tenantRiskCategories, categoryDefinitions, riskTemplates, industrySpecificRisks, methodologyRisks (read-only). Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: Feedback System:** Admin dashboard link "Feedback System". `/admin/feedback`: GET `/api/v1/admin/feedback-types` and GET/PUT `/api/v1/admin/feedback-config` via gateway (recommendations); lists feedback types (global); form for global feedback config (default limit, min/max limit, available types, default active types). Feedback aggregation section: GET `/api/v1/feedback/aggregation?period=daily|weekly|monthly` via gateway; period selector and Refresh; displays period, recommendations (shown, receivedFeedback, feedbackRate), sentiment (positive/neutral/negative), actionMetrics (intended, taken, rate), updatedAt. Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: Decision Rules:** Admin dashboard link "Decision Rules". `/admin/decision-rules`: GET `/api/v1/decisions/rules` via gateway (risk-analytics); table of rules (name, enabled, priority, conditions, catalogRiskId); per-rule "Test" opens modal with opportunityId input, POST `/api/v1/decisions/rules/:ruleId/test`, shows matched and actions count. Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: Action Catalog:** Admin dashboard link "Action Catalog". `/admin/action-catalog`: GET `/api/v1/action-catalog/entries` via gateway (risk-catalog); optional type filter (risk | recommendation); table of entries (type, name, category, status, version). Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: ML Models:** Admin dashboard link "ML Models". `/admin/ml-models`: GET `/api/v1/ml/models/health` via gateway (ml-service); displays overall status (ok/degraded/not_configured) and table of endpoints (status, latencyMs). Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: Feature Engineering:** Admin dashboard link "Feature Engineering". `/admin/feature-engineering`: GET `/api/v1/ml/features/versions` and GET `/api/v1/ml/features/schema` via gateway (ml-service); optional purpose filter; table of versions and schema (purpose, version, features). Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: Tenant Management:** Admin dashboard link "Tenant Management". `/admin/tenants`: GET `/api/v1/admin/tenants` via gateway (recommendations); table of tenants (id, name, industry, status, createdAt, activeUsers, activeOpportunities); View links to `/admin/tenants/[id]`; link to Tenant Templates. `/admin/tenants/[id]`: tenant detail with GET/PUT `/api/v1/admin/tenants/:tenantId/feedback-config`; full feedback config form (activeLimit, requireFeedback, allowComments, commentRequired, allowMultipleSelection, patternDetection, activeTypes add/remove from feedback-types). `/admin/tenants/templates`: GET `/api/v1/admin/tenant-templates` (list); Create template form (name, description, methodology, feedbackConfig including active types from feedback-types); Apply section: textarea for tenant IDs (comma or newline), POST `/api/v1/admin/tenant-templates/:templateId/apply` with `{ tenantIds }`; displays applied/failed result. Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: Analytics & Reporting (Plan §9):** Admin dashboard link "Analytics & Reporting". `/admin/analytics`: landing with links to Dashboards, Reports, Data Export. `/admin/analytics/dashboards`, `/admin/analytics/reports`, `/admin/analytics/export`: placeholder pages (builder/config to be implemented).
- **W11 – Super Admin: Security & Access Control (Plan §10):** Admin dashboard link "Security & Access Control". `/admin/security`: landing with links to Roles, Users, API Keys, Audit Log. `/admin/security/roles`: full page – Tenant ID input, GET `/api/v1/tenants/:tenantId/roles` via gateway (user_management); table (name, description, type, userCount, View); View links to `/admin/security/roles/[id]?tenantId=`. `/admin/security/roles/[id]`: placeholder role detail. `/admin/security/users`: full page – Tenant ID input, GET `/api/v1/users?tenantId=...` via gateway (user_management); displays member summary and members table (userId, email, name, role, status, joined). `/admin/security/api-keys`: Tenant ID input, "Load keys", "Create API key"; GET list, POST create (name, scope?, expiresAt?; raw key shown once with copy), DELETE revoke (confirm), POST rotate (confirm; new key shown once); table with Actions (Rotate, Revoke). Via gateway (user_management). `/admin/security/audit`: full page – GET `/api/logging/api/v1/logs` (list with filters: startDate, endDate, userId, action, category, severity, resourceType; limit/offset pagination), POST `/api/logging/api/v1/export` (request CSV export with same filters). Table: timestamp, user, action, resource, details, IP. Uses `NEXT_PUBLIC_API_BASE_URL` (no hardcoded ports).
- **W11 – Super Admin: System Configuration (Plan §8):** Admin dashboard card "System Configuration" (replaces separate Settings and Monitoring cards). `/admin/system`: landing with links to Performance (§8.1), Data Lake (§8.2), Logging (§8.3), API Security (§8.4), plus Settings and Monitoring (existing pages). `/admin/system/performance`, `/admin/system/data-lake`, `/admin/system/logging`, `/admin/system/api-security`: placeholder pages. Settings and Monitoring remain at `/admin/settings` and `/admin/monitoring`, linked from System landing.
- **Routes (Plan §889, §6.2, §932, §936):** `/dashboard`, `/dashboard/manager`, `/dashboard/executive` (ExecutiveDashboardPage; revenue-at-risk, forecast, competitive, RiskHeatmap, industry_benchmark stub; GET /api/v1/dashboards/executive), `/dashboard/board` (Plan §6.5, §932: BoardDashboardPage; revenue-at-risk, competitive win/loss, forecast; GET /api/v1/dashboards/board), `/opportunities`, `/opportunities/[id]`, `/opportunities/[id]/risk`, `/opportunities/[id]/remediation`, `/accounts/[id]`, `/analytics/competitive`, `/analytics/benchmarks`, `/analytics/portfolios` (Plan §6.5, §957: PortfolioDrillDownPage; DrillDownBreadcrumb; portfolio → account → opportunity → activity; data from GET /api/v1/portfolios/:id/summary, .../accounts; GET /api/v1/accounts/:id/opportunities; GET /api/v1/opportunities/:id/activities), `/settings/competitors`, `/settings/industries` (Plan §6.2, §6.5: IndustrySettingsPage; industry list and tenant enable/disable; stub until GET /api/v1/industries; link to benchmarks).
